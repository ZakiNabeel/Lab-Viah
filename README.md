## Executive Summary
RishtaAI is an **agentic AI matchmaking platform** designed for Pakistan’s informal rishta economy (aunties, baajis, marriage bureaus, WhatsApp groups, and phone-based discovery).

Instead of swipe-based apps, each user creates a persistent **AI Twin** through a short multimodal onboarding flow. Twins then negotiate compatibility across key dimensions of married life and surface only the **top 3 high-confidence matches** with transparent reasoning.

RishtaAI is intentionally framed as **halal matchmaking**, not dating.

## Core Product Pillars
1. **AI Twin Personality Engine**  
   Persistent per-user Twin created from a 3-layer hybrid onboarding:
   - Voice conversation
   - Scenario cards
   - Twin interview  
   Optional **Wali Mode** adds family/guardian input.

2. **Twin-to-Twin Negotiation**  
   A Moderator Agent orchestrates structured compatibility debate across 8 dimensions, producing explainable reports.

3. **Service Orchestration Layer (Challenge 2 aligned)**  
   Simulated end-to-end flows for:
   - Discovery & multilingual parsing with confidence scoring
   - Multi-factor matching
   - Dynamic pricing simulation
   - Booking/scheduling & confirmations
   - Follow-ups, dispute handling, and fallback workflows

4. **Halal Reveal + Wali Dashboard**  
   Top matches are revealed in a respectful flow prioritizing wali/family contact and Islamic adab.

## Antigravity Agentic Architecture (High-Level)
RishtaAI uses a roster of specialized agents coordinated by an orchestration layer:
- **Onboarding Agent(s)** to extract user preferences, values, and constraints
- **Twin Builder Agent** to maintain a stable personality/intent profile
- **Negotiation Moderator Agent** to run Twin-vs-Twin compatibility debate
- **Scoring Agent** to compute structured compatibility outcomes
- **Service Orchestrator Agent** for booking/pricing/scheduling simulation
- **Policy & Safety Agent** for halal framing, boundaries, and fallback logic
- **Dispute Agent** to model conflict-resolution paths
- **Explainability/Trace Agent** to expose workflow reasoning in demo output
